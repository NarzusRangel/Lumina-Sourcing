# Projects Organization

## Estrutura inicial sugerida
- Projeto 1: Pesquisa de Fornecedores | Uso geral comercial
- Projeto 2: Curadoria e Qualidade | Manutenção de padrão e exemplos
- Projeto 3: Operação Avançada | Casos que depois serão migrados para Codex ou integram memória operacional

## Critérios de organização
- Separar por público e objetivo operacional.
- Evitar projetos com regras concorrentes para o mesmo tipo de usuário.
- Reaproveitar a mesma base-mãe documental entre projetos.
