# Arquitetura ChatGPT Web

## Objetivo do canal
Oferecer a experiência mais simples para colaboradores comerciais que precisam pesquisar fornecedores ou estruturar uma demanda sem operar o framework completo.

## Papel do canal
- Capturar demandas simples ou medianas.
- Guiar a coleta de contexto mínimo.
- Entregar respostas úteis com baixo atrito.
- Encaminhar casos complexos para Codex.

## Limites do canal
- Não concentrar toda a complexidade operacional.
- Não assumir registro transacional avançado.
- Não carregar excesso de comandos ou terminologia técnica interna.

## Componentes previstos
- GPT instructions enxutas e consistentes.
- Knowledge files de contexto estável.
- Projects organizados por público, domínio ou tipo de uso.

## Critério de sucesso
Usuários gerais conseguem obter valor sem treinamento longo e sem depender de um mantenedor técnico.
