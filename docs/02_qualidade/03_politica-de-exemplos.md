# Política de Exemplos

## Regras
- Exemplo serve para calibrar comportamento, não para substituir regra.
- Todo exemplo deve explicitar cenário, motivo de acerto ou erro e lição operacional.
- Exemplos reais devem ser anonimizados quando necessário.
- Exemplo legado só entra se ajudar a explicar um padrão que será reescrito.

## Critérios de publicação
- O cenário precisa ser plausível.
- O exemplo ruim não pode ser caricato.
- A lição precisa ser reaproveitável.
- A origem do exemplo deve ser rastreável.
